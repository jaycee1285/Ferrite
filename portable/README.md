# Ferrite MD — PortableApps.com Packaging

This directory contains the [PortableApps.com Format (PAF)](https://portableapps.com/development/portableapps.com_format) packaging structure for distributing Ferrite as a portable application.

## How It Works

The PAF launcher (`FerriteMDPortable.exe`) sets the `FERRITE_DATA_DIR` environment variable to `Data\settings\` before launching `ferrite.exe`. Ferrite detects this variable and stores all configuration (settings, session, snippets) in that directory instead of `%APPDATA%`.

This means:
- **No files are written to the host system** (no registry, no AppData writes)
- Settings travel with the portable app on USB drives
- Fully compatible with the PortableApps.com Platform menu

## Directory Structure

```
FerriteMDPortable/
  FerriteMDPortable.exe          # PAF launcher (generated by PortableApps.com Launcher)
  help.html                      # -> copy from Other/Help/help.html
  App/
    AppInfo/
      appinfo.ini                # App metadata for PAF platform
      appicon.ico                # Multi-size icon (16/32/48/256)
      appicon_16.png             # 16x16 icon
      appicon_32.png             # 32x32 icon
      appicon_128.png            # 128x128 icon (optional but recommended)
      Launcher/
        FerriteMDPortable.ini    # Launcher configuration
    Ferrite/
      ferrite.exe                # The actual Ferrite binary (release build)
    DefaultData/
      settings/                  # Default config (empty — Ferrite creates defaults)
  Data/
    settings/                    # User data (config.json, session.json, snippets.json)
  Other/
    Source/
      LICENSE                    # MIT license
    Help/
      help.html                  # Help file
      Images/                    # Help images (if any)
```

## Build Steps

### Prerequisites

- [PortableApps.com Launcher](https://portableapps.com/apps/development/portableapps.com_launcher) (generates `FerriteMDPortable.exe`)
- [PortableApps.com Installer](https://portableapps.com/apps/development/portableapps.com_installer) (packages into `.paf.exe`)
- Python 3 + Pillow (`pip install Pillow`) for icon generation
- Rust toolchain for building Ferrite

### 1. Build Ferrite (release)

```bash
cargo build --release
```

### 2. Generate icons

```bash
cd assets/icons
python generate_all_icons.py <source-icon.png>
```

Copy the generated `portableapps/` contents into `portable/FerriteMDPortable/App/AppInfo/`:
- `appicon.ico`
- `appicon_16.png`
- `appicon_32.png`
- `appicon_128.png`

### 3. Copy the Ferrite binary

```bash
cp target/release/ferrite.exe portable/FerriteMDPortable/App/Ferrite/
```

### 4. Generate the PAF launcher

1. Download and extract the [PortableApps.com Launcher](https://portableapps.com/apps/development/portableapps.com_launcher)
2. Run `PortableApps.comLauncherGenerator.exe`
3. Point it to the `portable/FerriteMDPortable` directory
4. It will generate `FerriteMDPortable.exe` in the root of `FerriteMDPortable/`

### 5. Package with PAF Installer

1. Download and extract the [PortableApps.com Installer](https://portableapps.com/apps/development/portableapps.com_installer)
2. Run `PortableApps.comInstaller.exe`
3. Point it to the `portable/FerriteMDPortable` directory
4. Output: `FerriteMDPortable_0.2.7_English.paf.exe` (or Dev Test variant)

### 6. Test

1. Run the generated `.paf.exe` installer — it should extract cleanly
2. Launch `FerriteMDPortable.exe` from the installed location
3. Verify settings are stored in `Data/settings/` (not in `%APPDATA%`)
4. Close and relaunch — settings should persist
5. Delete `Data/settings/` and relaunch — defaults should be recreated

## Submission Process

1. **Dev Test release**: Name it `FerriteMDPortable_0.2.7_Dev_Test_1_English.paf.exe`
2. **Host the file** on GitHub Releases or SourceForge
3. **Post in the [Beta Testing forum](https://portableapps.com/forums/development/beta_testing)** with download link
4. **Iterate on feedback** from community testers
5. **Once approved**, post in the ["Apps Ready for Official Release" thread](https://portableapps.com/node/11772)

## Version Bumps

When releasing a new version:
1. Update `PackageVersion` and `DisplayVersion` in `App/AppInfo/appinfo.ini`
2. Copy the new `ferrite.exe` release build into `App/Ferrite/`
3. Re-run the Launcher Generator and Installer
