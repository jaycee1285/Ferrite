<task id="2" type="code">
  <name>Create worker module with generic pattern</name>
  <description>
    Create `src/workers/mod.rs` with:

    1. **WorkerCommand enum** - Generic command type
    2. **WorkerResponse enum** - Generic response type
    3. **WorkerHandle struct** - Holds mpsc sender/receiver
    4. **spawn_worker() function** - Launches tokio runtime in background thread

    Pattern to follow:
    ```rust
    use std::sync::mpsc::{self, Sender, Receiver};
    use std::thread;

    pub enum WorkerCommand {
        Echo(String),
        Shutdown,
    }

    pub enum WorkerResponse {
        Echo(String),
        Ready,
    }

    pub struct WorkerHandle {
        pub command_tx: Sender<WorkerCommand>,
        pub response_rx: Receiver<WorkerResponse>,
    }

    impl WorkerHandle {
        pub fn spawn<F>(worker_fn: F) -> Self
        where
            F: FnOnce(Receiver<WorkerCommand>, Sender<WorkerResponse>) + Send + 'static
        {
            let (cmd_tx, cmd_rx) = mpsc::channel();
            let (resp_tx, resp_rx) = mpsc::channel();

            thread::spawn(move || {
                worker_fn(cmd_rx, resp_tx);
            });

            Self {
                command_tx: cmd_tx,
                response_rx: resp_rx,
            }
        }
    }
    ```

    Key design:
    - std::sync::mpsc (NOT tokio mpsc - crosses thread boundary)
    - WorkerHandle::spawn() launches background thread
    - Worker function receives channels, runs tokio runtime inside
  </description>

<task id="3" type="code">
  <name>Create echo worker as proof-of-concept</name>
  <description>
    Create `src/workers/echo_worker.rs` that demonstrates the pattern:

    ```rust
    use super::{WorkerCommand, WorkerResponse};
    use std::sync::mpsc::{Receiver, Sender};

    pub fn echo_worker(cmd_rx: Receiver<WorkerCommand>, resp_tx: Sender<WorkerResponse>) {
        // Create tokio runtime in THIS thread (not main thread)
        let rt = tokio::runtime::Runtime::new().unwrap();

        // Send ready signal
        let _ = resp_tx.send(WorkerResponse::Ready);

        // Event loop
        loop {
            match cmd_rx.recv() {
                Ok(WorkerCommand::Echo(msg)) => {
                    // Simulate async work
                    let response = rt.block_on(async {
                        tokio::time::sleep(tokio::time::Duration::from_millis(100)).await;
                        format!("Echo: {}", msg)
                    });
                    let _ = resp_tx.send(WorkerResponse::Echo(response));
                }
                Ok(WorkerCommand::Shutdown) => break,
                Err(_) => break,
            }
        }
    }
    ```

    Export from mod.rs:
    ```rust
    mod echo_worker;
    pub use echo_worker::echo_worker;
    ```

    This proves:
    - Tokio runs in background thread (not main)
    - mpsc communication works
    - async operations don't block UI
  </description>
